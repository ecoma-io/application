## 🧩 i18n-service – Centralized Translation Manager

### 🎯 Mục tiêu:

Cung cấp khả năng lưu trữ, truy xuất và đồng bộ các bản dịch (language packs) theo từng app, version, feature và keyword. Hỗ trợ cache phía client và cập nhật realtime qua pub/sub.

---

## ⚙️ Database Schema

### `Translation`

```ts
Translation {
  _id: ObjectId;
  appName: string;
  appVersion: string;
  feature: string;
  keyword: string;
  language: string;
  value: string;
  verified: boolean;
  updatedAt: Date;
}
```

> **Uniqueness key:** `appName + appVersion + feature + keyword + language`

---

### `TranslationAudit`

```ts
TranslationAudit {
  _id: ObjectId;
  appName: string;
  appVersion: string;
  feature: string;
  keyword: string;
  language: string;
  oldValue: string | null;
  newValue: string;
  updatedBy: string; // worker hoặc userID
  source: "user" | "worker";
  verified: boolean;
  timestamp: Date;
}
```

> Lưu lại lịch sử thay đổi bản dịch phục vụ cho audit hoặc rollback nếu cần.

---

## ✅ Actions – i18n-service

### `i18n.register-language-pack`

- **Input:**

```ts
{
  appName: string;
  appVersion: string;
  defaultLanguage: string; // eg: "en"
  features: {
    [feature: string]: string[];
  };
}
```

- **Behavior:**
  - Lưu các từ khóa gốc (tiếng Anh) vào DB nếu chưa tồn tại.
  - Push từng từ khóa vào RabbitMQ để `i18n-worker` xử lý dịch tự động.

---

### `translation.update-translation`

- **Input:**

```ts
{
  appName: string;
  appVersion: string;
  feature: string;
  keyword: string;
  language: string;
  value: string;
  source?: "user" | "worker"; // default = "user"
  updatedBy: string;
}
```

- **Behavior:**
  - Nếu `source = "worker"`:
    - Nếu bản dịch đã `verified = true` → **bỏ qua, không ghi đè**.
    - Nếu chưa, cập nhật và để `verified = false`.
  - Nếu `source = "user"` → luôn cập nhật và `verified = true`.
  - Emit event:

```
[appName].[appVersion].translation-updated.[feature].[keyword]
```

Payload: `{ value: string }`

- Ghi log vào `TranslationAudit`

---

### `translation.get-language-pack`

- **Input:**

```ts
{
  appName: string;
  appVersion: string;
  language: string;
  feature?: string;
}
```

- **Output:** toàn bộ các bản dịch hiện có.

---

### `translation.get-latest-translation-by-language`

- **Input:**

```ts
{
  appName: string;
  keyword: string;
  feature: string;
  language: string;
}
```

- **Behavior:** tìm bản dịch gần nhất theo version mới nhất (nếu có), trả về `value | null`.

---

## 🧠 i18n-worker – Background Translator

### 🎯 Mục tiêu:

Xử lý dịch ngôn ngữ nền cho các từ khóa chưa có bản dịch chính thức.

---

### ✅ Workflow

1. **Consume language-pack từ RabbitMQ**

   - Chia nhỏ các `feature + keyword` → push từng task vào queue xử lý.

2. **Process từng task:**
   - Kiểm tra bản dịch tồn tại chưa:
     - Nếu đã có và `verified = true` → bỏ qua.
     - Nếu chưa:
       - Tìm bản dịch cũ nhất theo version trước đó → clone `value` nếu có.
       - Nếu không → dùng Translate API (Google, LibreTranslate)
   - Gọi `translation.update-translation` với:
     ```ts
     {
       ...,
       source: "worker",
       updatedBy: "i18n-worker"
     }
     ```

---

## 🛡️ Bảo vệ dữ liệu

- Worker **không được ghi đè** các bản dịch đã xác thực (`verified = true`).
- Chỉ user mới có thể cập nhật bản dịch có hiệu lực (`verified = true`).

---

## 📦 App-side Integration

### Hàm `_t()`

```ts
_t("Login");
_t("Hello {name}!", { name: "Alice" });
```

- Nếu chưa có bản dịch → hiển thị `keyword` (tiếng Anh gốc).
- Khi app khởi động:
  - Gửi `register-language-pack`
  - Gọi `get-language-pack` để cache
  - Subcribe các event `translation-updated` để update cache tự động.
